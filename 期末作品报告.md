## 1. 期末作品目的

本项目旨在综合运用微信小程序开发技术，完成一个具有实际应用价值的移动端应用。通过本项目的开发，达到以下目标：

2. **后端开发**：掌握 Node.js + Express + MySQL 的后端架构设计，实现 RESTful API 接口开发、JWT 认证、数据库设计等。
3. **全栈能力**：实现前端与后端的完整交互流程，包括用户登录、数据同步、实时通信等。
4. **项目管理**：使用 Git 进行版本控制，合理规划项目结构，按时完成开发任务。
5. **用户体验**：注重界面设计和交互逻辑，提供流畅、美观的用户体验。

## 3. 程序设计思路

### 3.1 整体架构

本项目采用前后端分离的架构设计：

```
┌─────────────────┐     HTTPS      ┌──────────────────┐     MySQL     ┌──────────┐
│  微信小程序前端   │ ────────────── │  Node.js 后端服务  │ ──────────── │  数据库   │
│  (WXML/WXSS/JS) │                │  (Express + JWT)  │              │ (MySQL)  │
└─────────────────┘                └──────────────────┘              └──────────┘
```

### 3.2 技术选型

**前端：**
- 微信小程序原生框架（WXML + WXSS + JavaScript）
- 微信开发者工具进行开发和调试
- 本地存储（wx.Storage）作为缓存层

**后端：**
- Node.js + Express 框架
- JWT（JSON Web Token）进行用户认证
- mysql2 连接 MySQL 数据库

**部署：**
- Railway 云平台部署后端服务
- Railway 托管 MySQL 数据库
- 微信公众平台配置服务器域名

### 3.3 数据库设计

数据库共 10 张表，设计如下：

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户表 | id, openid, nickname, avatar, discoverable |
| test_results | 测试结果表 | user_id, test_key, result_json |
| friendships | 好友关系表 | user_id, friend_id, note |
| achievements | 成就表 | user_id, achievement_id |
| points | 积分表 | user_id, xp, level, total_earned |
| point_history | 积分历史 | user_id, amount, reason |
| mood_history | 心情日记 | user_id, mood_data |
| daily_state | 每日一题 | user_id, streak, last_date, history_json |
| user_settings | 用户设置 | user_id, settings_json |
| challenges | 挑战表 | from_user_id, to_user_id, test_key, status, from_answers, to_answers, result |

### 3.4 核心流程设计

**用户登录流程：**
```
用户打开小程序 → wx.login() 获取 code → 发送到后端 → 后端用 code 生成 openid →
查找或创建用户 → 签发 JWT → 返回 token → 前端保存 token
```

**数据同步流程：**
```
本地操作 → 保存到 wx.Storage → 触发同步回调 → 异步发送到服务器 →
服务器更新数据库（前端不阻塞）
```

**远程挑战流程：**
```
用户 A 发起挑战 → 创建 challenge 记录（status=pending）→
用户 B 打开 app → 检测到待处理挑战 → 接受挑战（status=accepted）→
用户 B 答题提交 → 用户 A 答题提交 → 计算结果（status=completed）→
双方看到结果
```

### 3.5 安全设计

- 使用 JWT 进行身份认证，token 有效期 7 天
- 所有 API 接口使用参数化查询，防止 SQL 注入
- 用户 openid 不暴露给前端
- 积分接口限制单次增加上限（1-100）
- 昵称长度限制（最多 20 字符）
- 挑战答案格式校验
- CORS 跨域限制

## 5. 项目操作手册

### 5.1 环境准备

1. 安装微信开发者工具
2. 安装 Node.js（v16+）
3. 安装 MySQL（v8+）
4. 注册微信公众平台小程序账号

### 5.2 本地部署步骤

**步骤 1：克隆项目**
```bash
git clone https://github.com/blankLeaving99/weixinxiaochengxu.git
```

**步骤 2：初始化数据库**
```bash
mysql -u root -p < server/sql/init.sql
```

**步骤 3：配置后端**
```bash
cd server
npm install
```

修改 `server/config/db.js` 中的数据库连接信息：
```js
host: 'localhost',
user: 'root',
password: '你的密码',
database: 'personality_test'
```

**步骤 4：启动后端**
```bash
node app.js
```

看到 `🚀 性格测试服务器已启动: http://localhost:3001` 表示启动成功。

**步骤 5：配置小程序**

用微信开发者工具打开项目目录，在 `project.config.json` 中填入你的 AppID。

**步骤 6：运行**

在微信开发者工具中点击"编译"即可运行。

### 5.3 云端部署

后端服务部署在 Railway 云平台：
- 服务器地址：`https://weixinxiaochengxu-production.up.railway.app`
- 数据库：Railway 托管的 MySQL
- 环境变量：DB_HOST、DB_PORT、DB_USER、DB_PASSWORD、DB_NAME、JWT_SECRET、DEV_MODE

### 5.4 功能使用说明

**登录：** 打开小程序自动静默登录，无需手动操作。

**做测试：** 首页点击任意测试卡片 → 完成答题 → 查看结果。

**加好友：** 我的 → 好友管理 → 搜索添加好友 → 输入昵称或用户 ID。

**发起挑战：** 好友列表 → 点击"💕 挑战" → 选择"发送给好友"或"把手机交给对方"。

**备份恢复：** 我的 → 备份到云端 / 从云端恢复。

## 6. 小结（心得体会）

通过本次微信小程序项目的开发，我对移动开发技术有了更加深入和全面的理解。从最初的需求分析、技术选型，到数据库设计、前后端开发，再到云端部署和安全加固，整个过程让我深刻体会到了一个完整项目的开发流程。

在技术层面，我掌握了微信小程序的组件化开发模式，学会了 WXML 页面布局、WXSS 样式设计和 JavaScript 逻辑处理。特别是在实现远程挑战功能时，涉及到了轮询机制、状态管理和异步数据同步，这些都加深了我对前端异步编程的理解。后端方面，我学会了使用 Node.js + Express 构建 RESTful API，使用 JWT 实现无状态认证，使用 mysql2 进行数据库操作。这些都是课堂上学习过的知识，但只有在实际项目中应用，才能真正理解它们的原理和最佳实践。

在开发过程中，我遇到了很多问题。比如微信小程序不支持 `Object.entries()` 方法，导致页面报错；心情日记从服务器同步后日期字段丢失，导致周视图和月视图显示异常；积分恢复时使用累加而非覆盖，导致等级越恢复越高。这些问题让我学会了如何调试代码、分析日志、定位问题根源。我也深刻认识到，开发不仅仅是写代码，更重要的是测试、调试和优化。

在项目管理方面，我使用 Git 进行版本控制，养成了频繁提交、写好 commit message 的习惯。当代码出现问题时，可以通过 git log 回溯历史，快速定位引入问题的提交。这对于团队协作和项目维护非常重要。

在安全性方面，我学习了如何防范 SQL 注入、如何保护用户敏感信息（如 openid）、如何限制接口参数防止刷积分等。这些安全意识对于一个合格的开发者来说是必不可少的。

最后，感谢吴献老师的悉心指导。这门课程不仅教会了我移动开发的技术知识，更培养了我独立思考、解决问题的能力。在未来的学习和工作中，我会继续深入学习移动开发技术，不断提升自己的编程水平和项目实战能力。
